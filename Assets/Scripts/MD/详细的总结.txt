* 选题的背景及意义 共输入 672 字符

近年来，随着《Hades》、《死亡细胞》等游戏的成功，Roguelike（肉鸽）元素与动作角色扮演（ARPG）结合的“Action Roguelike”品类已成为游戏行业的重要发展方向。该类游戏强调高频率的战斗交互、随机生成的关卡体验以及深度的角色构建（Build）系统。然而，开发一款高质量的3D动作游戏面临着诸多技术挑战，包括复杂的动作状态管理、高性能的渲染需求以及高耦合的代码架构维护问题。

Unity作为当前主流的游戏开发引擎，虽然降低了入门门槛，但在处理中大型复杂系统时，传统的MonoBehaviour开发模式容易导致代码耦合度过高，难以扩展。因此，探索基于现代软件工程思想（如依赖注入、数据驱动）的游戏开发流程，具有重要的实践价值。

工程价值：本项目旨在探索如何在Unity环境中应用**依赖注入（Dependency Injection）**架构（使用VContainer）来解耦游戏逻辑与数据，解决传统开发模式中单例滥用和组件强依赖的问题，为独立游戏开发提供一套可维护性高的代码架构参考。

技术实践：通过实现**有限状态机（FSM）管理角色动作与行为树（Behavior Tree）驱动敌人AI，深入研究游戏人工智能与复杂状态流转的实现机制；同时应用URP（通用渲染管线）**技术，探索在有限性能开销下实现卡通渲染与粒子特效（VFX）的最佳实践。

应用前景：本项目最终将产出一款具备完整核心循环的独立游戏Demo，不仅是对大学期间所学计算机图形学、软件工程、数据结构等知识的综合应用，也是面向游戏客户端开发岗位的具备竞争力的求职作品。

* 工作目标及内容 共输入 1021 字符

1. 工作目标

设计并实现一款具备二次元风格的3D高速动作Roguelike游戏。游戏需包含流畅的角色战斗系统、智能的敌对生物AI、随机的增益强化（海克斯）系统以及完善的关卡循环。系统需运行稳定，帧率在主流PC上保持60FPS以上，代码结构需符合面向对象设计原则（SOLID）。

2. 研究内容

本课题的主要研究内容涵盖以下四个模块：

底层架构搭建：

引入VContainer 框架构建全局依赖注入环境，统一管理系统生命周期。

基于 UniTask 实现全异步的资源加载与场景流转。

搭建基于发布/订阅模式的事件总线，实现各功能模块间的解耦。

核心战斗系统设计（The 3C）：

底层移动框架搭建：基于 KinematicCharacterMotor 实现底层的物理碰撞与位移解算，解决斜坡滑落、台阶吸附等基础物理问题。

高级状态机设计：设计并实现基于类的有限状态机（Class-based FSM），将角色的行为拆分为 DefaultState（地面/空中）、ClimbingState（攀爬）等，替代传统的枚举判断，实现逻辑的模块化。

智能相机系统：集成 Cinemachine 插件，实现 World Space 模式的第三人称自由视角（FreeLook）。开发自定义脚本实现视觉无损的滚轮等比缩放功能，解决低视角下的地面碰撞抖动问题。

事件驱动架构：构建 EventManager 事件总线，实现 PlayerInputSystem（输入层）与 KCCMoveController（控制层）的完全解耦，利用结构体 KCCInputs 进行数据包传输。

手感调优与平滑处理：研究 FixedUpdate 与 LateUpdate 的执行时序，通过开启 KCC 插值（Interpolation）与分离视觉/物理旋转逻辑，解决高刷新率显示器下的画面抖动与残影问题。

Roguelike 玩法系统：

开发“海克斯”强化系统，支持数值修正与机制Hook（如改变攻击模组）。

基于 CSV 与 Odin Inspector 构建数据驱动的配置流程，实现技能、道具、敌人属性的可视化配置。

设计关卡波次管理与商店经济循环系统。

AI 与 视觉表现：

基于代码驱动的行为树（Behavior Tree）与黑板（Blackboard）模式，设计具备多阶段转换能力的Boss AI。

使用URP管线，结合ShaderGraph制作卡通渲染材质，利用VFXGraph实现高性能的技能特效与弹幕。


* 拟采用方法和步骤或设计思路 共输入 683 字符

三、 拟采用方法和步骤或设计思路

1. 设计思路

架构优先： 摒弃“面条式代码”，采用控制反转（IoC）思想。核心逻辑层不依赖于Unity的MonoBehaviour生命周期，而是通过纯C#类进行逻辑封装，仅在表现层与引擎API交互。

数据驱动： 游戏内的所有数值（攻击力、冷却、掉落率）均通过外部CSV表格管理，程序只负责解析与逻辑运算，实现策划数据与程序逻辑的分离。

脑体分离： 在AI设计中，将决策逻辑（行为树/大脑）与执行逻辑（状态机/身体）彻底分离，提高代码的复用性。

2. 拟采用方法

开发工具： Unity 2022 LTS (IDE), JetBrains Rider (代码编辑), Blender/Mixamo (资产处理), Git (版本控制)。

关键技术： VContainer (DI框架), Universal Render Pipeline (渲染), Native FSM (状态管理), Fluent Behavior Tree (AI), YooAsset (资源管理)。

3. 开发步骤

原型阶段： 搭建VContainer环境，实现胶囊体的移动、攻击与基础FSM状态切换。

核心战斗阶段： 接入动画资源，打磨连招手感，实现攻击判定（Hitbox）与受击反馈。

系统填充阶段： 开发CSV解析器，实现随机Buff（海克斯）系统、敌人波次刷新与商店逻辑。

AI与内容阶段： 构建行为树框架，实现Boss的复杂行为逻辑（阶段转换、技能释放）。

表现与优化阶段： 升级URP管线，制作VFX特效，引入对象池（Object Pool）优化性能，进行Bug修复。
