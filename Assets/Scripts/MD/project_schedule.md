# 📅 游戏开发计划 (3月16日 - 4月20日)

**目标**: 适配最终需求规格书，完成《Dream》核心玩法循环与Roguelite要素，并完成整体体验打磨。
**工作节奏**: 每天约 4 小时有效开发时间。
**总天数**: 36天 (预留充足打磨与修Bug时间)

---

## Phase 1: 战斗收尾与数值配置贯通
**时间**: 3月16日 - 3月20日 (5天)
**目标**: 彻底打通已有代码与配置表，建立最基础的战斗互动。
* **3.16** - `CSVHelper` 与现有 `Config` 类的完全打通与实装测试 (角色属性、Hex、怪物数据等)。
* **3.17** - NodeCanvas实装：敌人检测玩家然后使用NavMeshSurface寻路 (无障碍物场地)。
* **3.18** - 敌人 `Attack` 行为与伤害判定 (HitBox接入)。
* **3.19** - 玩家受击(`HitState`)、生命值扣除判定、死亡状态与简版死亡逻辑。
* **3.20** - **里程碑测试**：玩家能自由攻击，小怪能追踪并攻击玩家，完成双向伤害闭环。

---

## Phase 2: 关卡流程与跨场景循环
**时间**: 3月21日 - 3月24日 (4天)
**目标**: 实现局内外流转和基本的波次刷怪机制。
* **3.21** - 场景框架搭建：大厅选人 (MainMenu) 与 战斗关卡场景。
* **3.22** - 跨场景数据持久化：`GameSessionData` 纯 C# 类与 VContainer 全局注入。
* **3.23** - `LevelManager` 与 `WaveData` 逻辑：完成配置每波 6 只怪，死一补一，管理对象池。
* **3.24** - **里程碑测试**：大厅 -> 进入战斗 -> 按照波次刷怪打完 2 波 -> 结算返回大厅。

---

## Phase 3: UI 与 HUD (战斗表现)
**时间**: 3月25日 - 3月27日 (3天)
**目标**: 战斗过程中的必要信息展示。
* **3.25** - 玩家血本 UI 与状态栏变化（跟随受伤即时扣减）。
* **3.26** - 敌人头顶血条，基于世界坐标到屏幕空间的随动。
* **3.27** - 伤害飘字池化组件 (`DamageNumberPool`)与飘字特效动画实装联调。

---

## Phase 4: Roguelite 基础与道具掉落
**时间**: 3月28日 - 3月31日 (4天)
**目标**: 搭建局内成长的核心机制框架。
* **3.28** - `Buff/Modifier` 系统接入：完成玩家基础数值的动态增减益（攻、血、防加成）。
* **3.29** - 怪物死亡掉落逻辑：金币与基础数值道具拾取。
* **3.30** - 小关结算与 `HexSelectionUI` 三选一界面：暂停游戏并弹出 UI。
* **3.31** - 选取 Hex 后，触发能力升级指令并返回游戏。

---

## Phase 5: 大核内容堆量 (Boss、海克斯、道具)
**时间**: 4月1日 - 4月8日 (8天)
**目标**: 消耗精力最多的量产环节，依靠填写配置表驱动内容 (每天4小时主力产出)。
* **4.01 - 4.02** - 制作第一大关 Boss 预制体、导入模型、挂载 HitBox 调整手感。
* **4.03 - 4.04** - Boss 行为树：基于 NodeCanvas 配置特有动作和分阶段（半血转阶段二狂暴）。
* **4.05 - 4.07** - **海克斯/道具量产**：依据配置表大批量派生 Modifier 与特殊 Buff 类 (攻击附带特效等)。
* **4.08** - **里程碑综合测试**：一个完整大关卡 (3小关 + 1 Boss) 及其连带局内 Hex 强化流派顺畅跑通。

---

## Phase 6: 表现打磨与动作反馈
**时间**: 4月9日 - 4月12日 (4天)
**目标**: 提升 "高速动作战斗" 的爽快感与打击感。
* **4.09** - Hitstop (顿帧) 机制实现，加强挥砍力量感。
* **4.10** - 屏幕震动 (Camera Shake) 组件与受伤闪白效果集成。
* **4.11** - 命中特效 (VFX) 与武器攻击拖尾补齐。
* **4.12** - 音效系统接入 (挥击、命中被弹、受击、UI 交互)。

---

## Phase 7: QA 测试与数值平衡 (关键缓冲期)
**时间**: 4月13日 - 4月18日 (6天)
**目标**: 整体联调与难度体验调节。
* **4.13** - 完整大循环黑盒体验测试，记录阻断性和表现类 Bug。
* **4.14 - 4.15** - 集中修复核心逻辑和 UI 显示错误。
* **4.16 - 4.17** - 数值平衡调整：玩家整体伤害成长曲线、敌人血量曲线及难易度调参。
* **4.18** - 性能粗查 (Profiler)：关注战斗中对象实例化的卡顿，完善 Object Pool 复用。

---

## Phase 8: 冲刺与打包封版
**时间**: 4月19日 - 4月20日 (2天)
**目标**: 产出稳定的可执行文件。
* **4.19** - 最后一轮回归测试，确保在 Windows 环境下不崩溃。
* **4.20** - **里程碑：封板日**。Release Build 完成并打包。
